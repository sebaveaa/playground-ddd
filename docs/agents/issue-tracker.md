# Issue tracker: Linear

Issues and PRDs for this repo live in Linear. Use Linear as the source of truth for planning, triage, and work tracking.

Primary backlog project:

- [UMBRAL MVP Backlog](https://linear.app/umbralsv/project/umbral-mvp-backlog-06582636352e)

## Conventions

- Create issues in Linear rather than GitHub or local markdown.
- Read and update the relevant Linear issue when a skill needs ticket context.
- Apply the triage labels/status vocabulary from `docs/agents/triage-labels.md` using the closest matching Linear labels, workflow states, or custom fields.
- When a skill says "publish to the issue tracker", create a Linear issue.
- When a skill says "fetch the relevant ticket", open the corresponding Linear issue and use its description, labels, and comments as the current source of truth.

## Notes for skills

This repo does not use GitHub Issues as its primary tracker. If a skill normally assumes `gh issue ...`, adapt that workflow to Linear instead.
